## Project Overview

This project analyses 878,561 hotel reviews sourced from a hotel review JSON file. The pipeline:

1. **Cleans and normalises** raw JSON reviews into a structured SQLite database
2. **Explores** rating distributions, temporal trends, and reviewer behaviour
3. **Segments** hotels into 5 competitive clusters via K-means clustering
4. **Benchmarks** query and data-loading performance against the SQLite database
5. **Visualises** insights in a real-time Streamlit dashboard

**Key outputs:**
- `data/reviews_analysis.db` — 80,000 reviews with hotel/author aggregates (109 MB)
- `data/reviews_sample.db` — 6,000-review sample for fast iteration (8.4 MB)
- 5 named hotel segments: *Top of the Line*, *Convenient*, *Below Average*, *Terrible*, *Good but Inconvenient*

## Data

### Raw Source

`data/review.json` — 878,561 hotel reviews, newline-delimited JSON. Each record contains:
- Review text, title, and date
- 9 sub-ratings: service, cleanliness, overall, value, location, sleep quality, rooms, check-in, business service
- Author profile (username, location, review/vote counts)
- Hotel identifier

Both SQLite databases share the same schema (see [Database Schema](#database-schema)):

| `reviews_sample.db` | 6,000 | ~6,000 | ~1,500 | 8.4 MB |

Use `reviews_sample.db` for rapid development; `reviews_analysis.db` for full analysis.

## Notebooks

Run notebooks in order. Each notebook reads from sources produced by the previous one.

### 01 — Data Preparation

**Input:** `data/review.json`
**Output:** `data/reviews_analysis.db`, `data/reviews_sample.db`

- Normalises nested ratings into flat columns
- Deduplicates authors and resolves aliases
- Filters to the most recent 5 years of reviews
- Retains only reviews posted within 1 month of the stay date (84.75% retention)
- Removes 38 content-duplicate reviews
- Writes three tables (`reviews`, `authors`, `hotels`) with 8 performance indexes

### 02 — Exploratory Analysis

**Input:** `data/reviews_analysis.db`

- Rating distribution histograms across all 9 dimensions
- Temporal trends: review volume by year and season
- Correlation heatmaps between rating dimensions
- Author behaviour: reviews per author, helpful vote distributions
- Mobile vs. desktop review patterns

Key finding: location ratings are highest (mean ~4.4/5) and least correlated with other dimensions; service, cleanliness, and rooms are most intercorrelated.

### 03 — Competitive Benchmarking

**Input:** `data/reviews_analysis.db`
**Output:** `kmeans_cluster`, `PCA1`, `PCA2` columns written back to `hotels` table

- Clusters 3,380 hotels on service, location, and rooms ratings using K-means (k=5)
- Selects k via elbow method and silhouette analysis (score: 0.336)
- Visualises clusters in 2D via PCA

| Cluster | Label | Size | Strategy |
|---|---|---|---|
| 0 | Top of the Line | 210 (6%) | Maintain excellence, market reviews |
| 1 | Convenient | 1,257 (37%) | Lean into location marketing |
| 2 | Below Average | 490 (14%) | Prioritise service and rooms upgrades |
| 3 | Terrible | 666 (20%) | Emergency management overhaul |
| 4 | Good but Inconvenient | 757 (22%) | Offer transport solutions |

### 04 — Performance Profiling

**Input:** `data/reviews_analysis.db`

- Compares data-loading strategies: full load vs. chunked vs. column projection
  - Column projection (6 columns) is 2.8× faster than `SELECT *`
- Benchmarks 6 SQL queries with and without indexes
- Analyses index selectivity and covering-index effectiveness
- Profiles K-means scalability (time vs. k)

Results and charts are saved to `profiling/`.

## Streamlit Dashboard

The dashboard connects to `data/reviews_analysis.db` and provides three tabs per hotel:

| Tab | Contents |
|---|---|
| **Executive Summary** | KPIs (overall rating, review count, satisfaction rate), monthly rating trend, radar chart of quality dimensions, operational alerts |
| **Customer Voice** | Filterable review table (all / positive / negative) with search |
| **Benchmarking** | Cluster label and strategy, PCA scatter plot, market comparison bar chart |

### Running the Dashboard

```bash
streamlit run app/streamlit_app.py
```

Then open [http://localhost:8501](http://localhost:8501) in your browser. Select a hotel from the sidebar to explore its analytics.

> **Note:** The dashboard requires `data/reviews_analysis.db`. Run notebook `01` first if it does not exist.

## Database Schema

```sql
reviews (
    review_id TEXT PRIMARY KEY,
    title TEXT,
    text TEXT,
    date_stayed TIMESTAMP,
    review_date TIMESTAMP,
    num_helpful_votes INTEGER,
    via_mobile INTEGER,
    service_rating REAL,
    cleanliness_rating REAL,
    overall_rating REAL,
    value_rating REAL,
    location_rating REAL,
    sleep_quality_rating REAL,
    rooms_rating REAL,
    check_in_service_rating REAL,
    business_service_rating REAL,
    hotel_id TEXT,
    author_id TEXT
)

authors (
    id TEXT PRIMARY KEY,
    username TEXT,
    location TEXT,
    alias TEXT,
    num_cities REAL,
    num_helpful_votes REAL,
    num_reviews REAL,
    num_type_reviews REAL
)

hotels (
    hotel_id TEXT PRIMARY KEY,
    num_reviews INTEGER,
    avg_service_rating REAL,
    avg_cleanliness_rating REAL,
    avg_overall_rating REAL,
    avg_value_rating REAL,
    avg_location_rating REAL,
    avg_sleep_quality_rating REAL,
    avg_rooms_rating REAL,
    avg_check_in_service_rating REAL,
    avg_business_service_rating REAL,
    -- added by notebook 03:
    kmeans_cluster INTEGER,
    PCA1 REAL,
    PCA2 REAL
)
```

**Indexes:** `hotel_id`, `review_date`, `(hotel_id, review_date)`, and unique constraints on all primary keys.
